> 장애 대응에서 기능 개발, 보안 조치, 인프라 개선까지 끝까지 가져가는 백엔드 개발자입니다.

`Java` `Spring Boot` `JPA` `MySQL` `Linux` `Tomcat` `Apache` `Git` `API` `운영 자동화`

---

## 한 줄로 보는 강점

**레거시와 운영 환경을 읽고, 실제 서비스가 멈추지 않게 개선하는 개발자입니다.**

넷매니아(주)에서 전자세금계산서 솔루션을 운영하며 API, 에이전트, 어드민, 인증서, 홈택스 스크래핑, 서버 설정, 보안 점검까지 폭넓게 다뤘습니다. 단순 유지보수에 머무르지 않고, 반복 문의와 장애 원인을 구조적으로 정리해 기능 개선과 문서화로 연결했습니다.

---

## 대표 임팩트

### 1. 전자세금계산서 핵심 도메인 운영

- 세금계산서 발행, 역발행, 매입/매출 보관함, 국세청 전송, 휴폐업 조회 등 전자세금계산서 업무 흐름을 운영 관점에서 분석하고 개선했습니다.
- 고객사별 API 발행 누락, 신고 실패, 인증서 오류, 동기화 오류를 추적해 데이터와 프로세스 기준으로 원인을 좁혔습니다.
- 대량발행, 문서 재서명, 재동기화, 지연전송 트리거 등 운영 리스크가 큰 업무를 처리하며 서비스 안정성을 유지했습니다.

### 2. AI 세금계산서 발행 기능 개발

- AI 세금계산서 발행 기능의 프론트 개발, QA 체크리스트 작성, 운영팀 기능 안내, 모바일 테스트까지 진행했습니다.
- 발행 모니터링, 리팩토링, 추가 기능 반영을 반복하며 신규 기능을 운영 가능한 수준으로 다듬었습니다.
- 신규 입사자에게 AI 세금계산서 발행 프론트 업무를 인수인계할 수 있도록 흐름을 정리했습니다.

### 3. API와 에이전트 문제 해결

- 센드빌 API, 휴폐업 API, 요금조회 API, 대량발행 API, 에이전트 연동 문의를 분석하고 가이드와 개선 작업으로 연결했습니다.
- 에이전트 DB 개인정보 암호화 로직 오류, TLS 1.2 전환, XML 평문 전송 옵션, Oracle/MySQL 드라이버 이슈 등 연동 환경별 문제를 다뤘습니다.
- 고객사 서버 이전, 신규 서버 에이전트 셋업, 에이전트 재설치/자동실행/로그인/전송 오류를 지원하며 운영 지식을 축적했습니다.

### 4. 보안과 인프라 개선

- 웹 취약점 점검 결과를 바탕으로 LOW 레벨 취약점, DNS 보안 사항, SQL Injection 대응, Apache/Tomcat 버전 영향도 분석을 수행했습니다.
- API 및 센드빌 SSL 인증서 교체, DCV DNS TXT 적용, 서버 다운 시 포워딩 오류 페이지 개발을 진행했습니다.
- NFS, 스토리지 풀, 리눅스 마운트, 서버 볼륨 포맷, 웹/WAS 분리 과정 문서화 등 운영 인프라를 직접 확인하고 정리했습니다.

### 5. 어드민과 운영 도구 개선

- 어드민 발행현황, 회원현황 그래프, 연동서비스 화면, 통계 SMS, 주간 발행량 SMS 스케줄러를 개발하거나 개선했습니다.
- 휴폐업 SQL 쿼리 최적화, 회원현황 쿼리 튜닝, 거래처 요금제 정보 쿼리 수정 등 운영 화면의 정확도와 속도를 개선했습니다.
- 어드민 외부 IP 차단, 크롤링 방지, OTP 문의 대응 등 관리자 도구의 보안성과 운영성을 함께 챙겼습니다.
